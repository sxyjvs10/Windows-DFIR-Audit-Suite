[!] INCIDENT SUMMARY
--------------------
A cryptocurrency miner was found bundled inside a cracked/pirated Adobe
Creative Cloud installation (Photoshop 2021/2023, After Effects 2022,
Premiere Pro). The miner used "Process Hollowing" — hiding inside legitimate
Adobe process names so Task Manager showed normal-looking processes instead
of a miner executable.

The miner ran 24/7 in the background, consuming CPU/GPU/RAM to mine
cryptocurrency (likely Monero/XMR) for the attacker, while the victim
experienced severe system slowdowns and application hangs.

Attack Vector  : Cracked Adobe software downloaded from unofficial source
Miner Type     : Process Hollowing (disguised inside Adobe processes)
Crypto Mined   : Likely Monero (XMR) — CPU-mineable, untraceable
Impact         : 73.5% RAM usage, system hangs, thermal throttling, slow PC


================================================================================
[1] HOW THE INFECTION OCCURRED
================================================================================

STEP 1 — User downloaded cracked Adobe software from an unofficial site.

STEP 2 — The crack installer ran silently and did two things:
          a) Installed Adobe apps normally (so user suspects nothing)
          b) Bundled a miner inside Adobe's own processes (process hollowing)

STEP 3 — The installer modified the Windows HOSTS file to block Adobe's
          license verification servers, so the cracked app kept working.

STEP 4 — The miner ran 24/7 hidden as legitimate Adobe process names:
          - Adobe Desktop Service.exe
          - CoreSync.exe
          - AdobeUpdateService.exe

STEP 5 — The attacker earns crypto. The victim gets a slow, hot, laggy laptop.


================================================================================
[2] PERSISTENCE MECHANISMS (How It Survived Reboots)
================================================================================

The miner used 6 layers of persistence to ensure it restarted automatically
after every reboot. Removing any one layer was not enough — all had to be
eliminated simultaneously.

--------------------------------------------------------------------------------
LAYER 1 — Windows Service (Primary Persistence)
--------------------------------------------------------------------------------
What       : "Adobe Desktop Service" registered as an Automatic Windows Service
Location   : C:\Program Files (x86)\Common Files\Adobe\Adobe Desktop Common\
             ADS\Adobe Desktop Service.exe
Registry   : HKLM\SYSTEM\CurrentControlSet\Services\AdobeUpdateService
Behavior   : Starts automatically before user login (SYSTEM level privileges)
Risk Level : HIGH — Starts even before the desktop loads

How to Detect:
  > Get-Service | Where-Object {$_.Name -like '*adobe*'}
  > sc query AdobeUpdateService

--------------------------------------------------------------------------------
LAYER 2 — Shell Extension DLL (Stealth Persistence)
--------------------------------------------------------------------------------
What       : CoreSync_x64.dll registered as a Windows Shell Extension
Location   : C:\Program Files (x86)\Common Files\Adobe\CoreSyncExtension\
             CoreSync_x64.dll
Registry   : HKLM\SOFTWARE\Microsoft\Windows\CurrentVersion\Shell Extensions\
             Approved\{CLSID of CoreSync}
Behavior   : DLL automatically loaded INTO Explorer.exe on every login.
             Because it lives inside Explorer's memory space, it is extremely
             hard to detect and impossible to delete while Explorer is running.
Risk Level : CRITICAL — Loaded inside Explorer = survives process kill attempts

How to Detect:
  > Sysinternals Autoruns → Shell Extensions tab
  > reg query "HKLM\SOFTWARE\Microsoft\Windows\CurrentVersion\Shell Extensions\Approved"

--------------------------------------------------------------------------------
LAYER 3 — Registry Autoruns
--------------------------------------------------------------------------------
What       : Adobe keys in Run and software registry hives
Locations  :
  HKCU\Software\Adobe\                  (user-level config + autoruns)
  HKLM\SOFTWARE\Adobe\                  (system-level config, 64-bit)
  HKLM\SOFTWARE\WOW6432Node\Adobe\      (system-level config, 32-bit on 64-bit OS)

Behavior   : Stores miner configuration including:
             - Mining pool address (e.g., pool.minexmr.com:4444)
             - Attacker's wallet address
             - Autorun triggers
             - COM object registration for shell hooks
Risk Level : HIGH — Both 32-bit and 64-bit registry hives must be cleaned

How to Detect:
  > reg query HKLM\SOFTWARE\Adobe
  > reg query HKLM\SOFTWARE\WOW6432Node\Adobe
  > Sysinternals Autoruns → Everything tab, filter by Adobe

--------------------------------------------------------------------------------
LAYER 4 — HOSTS File Modification (Evasion + Persistence Aid)
--------------------------------------------------------------------------------
What       : C:\Windows\System32\drivers\etc\hosts was modified to redirect
             Adobe's license/telemetry servers to 127.0.0.1 (localhost)

Modified entries found:
  127.0.0.1  genuine.adobe.com
  127.0.0.1  prod.adobegenuine.com
  127.0.0.1  cc-api-data.adobe.io
  127.0.0.1  ic.adobe.io
  127.0.0.1  3ca52znvmj.adobe.io
  127.0.0.1  assets.adobedtm.com

Purpose    :
  a) Prevent Adobe from detecting the cracked/invalid license
  b) Block Adobe telemetry that could report the crack/modification
  c) (Advanced) Can also block AV update servers if additional entries added

Risk Level : MEDIUM — Evasion mechanism, not direct miner persistence
             But CRITICAL indicator — always check hosts file during IR

How to Detect:
  > Get-Content C:\Windows\System32\drivers\etc\hosts
  > Look for any non-comment, non-localhost entries
  > Any redirects to 127.0.0.1 for known domains = RED FLAG

--------------------------------------------------------------------------------
LAYER 5 — AppData Folders (Configuration Storage)
--------------------------------------------------------------------------------
What       : Miner's working directory — stores config, logs, and wallet data
Locations  :
  C:\Users\sayoo\AppData\Roaming\Adobe\             (main config)
  C:\Users\sayoo\AppData\Roaming\com.adobe.dunamis\ (IPC mechanism)
  C:\Users\sayoo\AppData\Local\Adobe\               (cache + temp output)

Contents   :
  - Mining pool URL and port
  - Attacker wallet address (XMR/crypto address)
  - Mining threads configuration (how many CPU threads to use)
  - Log files of mining activity
  - Encrypted communication keys

Risk Level : MEDIUM — Secondary persistence (config survives if binary missed)
             If config survives, miner can re-download itself if re-triggered

How to Detect:
  > Get-ChildItem $env:APPDATA\Adobe -Recurse | Select FullName, LastWriteTime
  > Look for recently modified files especially .json, .conf, .dat files

--------------------------------------------------------------------------------
LAYER 6 — In-Memory Residency
--------------------------------------------------------------------------------
What       : Even after disk files are deleted, loaded DLLs remain in RAM
             The miner's code running inside Explorer.exe/svchost.exe persists
             in memory until the process is killed or system rebooted.

Risk Level : LOW (temporary) — Cleared automatically on reboot

How to Detect:
  > Sysinternals Process Explorer → right-click process → Find DLLs
  > PE-sieve tool: pe-sieve.exe --pid <PID> (detects hollowed processes)
  > Hollows Hunter: hollows_hunter.exe (automated hollowing detection)

[4] REMOVAL METHODOLOGY
================================================================================

The removal had to follow a strict order because of the lock chain:
"Can't delete what you don't own, can't own what's locked"

STEP 1 — KILL ACTIVE PROCESSES
  Reason  : Release file handles so files can be deleted
  Command :
    Get-Process | Where-Object { $_.Name -like '*adobe*' } | Stop-Process -Force

STEP 2 — KILL EXPLORER.EXE
  Reason  : CoreSync_x64.dll was loaded INTO Explorer's memory
            Explorer held an active handle on the DLL = file locked
            Killing Explorer = DLL unloaded = handle released = deletable
  Command :
    taskkill /f /im explorer.exe

  WARNING : Desktop and taskbar disappear — this is expected behavior.
            Must restart Explorer after deletion.

STEP 3 — TAKE OWNERSHIP (TrustedInstaller bypass)
  Reason  : Files under C:\Program Files are owned by TrustedInstaller
            Even administrators cannot delete TrustedInstaller-owned files
            Must transfer ownership to current user first
  Command :
    takeown /f "C:\Program Files (x86)\Common Files\Adobe" /r /d y
    takeown /f "C:\Program Files\Adobe" /r /d y
    takeown /f "C:\Program Files (x86)\Adobe" /r /d y
    takeown /f "C:\Program Files\Common Files\Adobe" /r /d y

  What TrustedInstaller is:
    A special Windows service account with higher privileges than SYSTEM
    Used to protect Windows component files from accidental/malicious deletion
    Malware installs files under this context to make removal harder

STEP 4 — GRANT FULL PERMISSIONS
  Reason  : Ownership alone is not enough — ACL (Access Control List) also
            needs to grant the current user write/delete permissions
  Command :
    icacls "C:\Program Files (x86)\Common Files\Adobe" /grant Everyone:F /t /c /q
    icacls "C:\Program Files\Adobe" /grant Everyone:F /t /c /q
    icacls "C:\Program Files (x86)\Adobe" /grant Everyone:F /t /c /q
    icacls "C:\Program Files\Common Files\Adobe" /grant Everyone:F /t /c /q

  Flags explained:
    /grant Everyone:F  → Grant Full Control to Everyone
    /t                 → Apply recursively to all subfolders and files
    /c                 → Continue on error (skip undeletable files)
    /q                 → Quiet mode (suppress output)

STEP 5 — DELETE PROGRAM FILES
  Command :
    Remove-Item -Path "C:\Program Files\Adobe" -Recurse -Force
    Remove-Item -Path "C:\Program Files (x86)\Adobe" -Recurse -Force
    Remove-Item -Path "C:\Program Files (x86)\Common Files\Adobe" -Recurse -Force
    Remove-Item -Path "C:\Program Files\Common Files\Adobe" -Recurse -Force

STEP 6 — DELETE APPDATA (User-owned, no TrustedInstaller needed)
  Command :
    Remove-Item "$env:APPDATA\Adobe" -Recurse -Force
    Remove-Item "$env:APPDATA\com.adobe.dunamis" -Recurse -Force
    Remove-Item "$env:LOCALAPPDATA\Adobe" -Recurse -Force

STEP 7 — CLEAN REGISTRY
  Reason  : Even with binaries gone, registry keys can re-register services
            or COM objects if not cleaned. Must clean ALL hives.
  Command :
    Remove-Item "HKCU:\Software\Adobe" -Recurse -Force
    reg delete "HKLM\SOFTWARE\Adobe" /f
    reg delete "HKLM\SOFTWARE\WOW6432Node\Adobe" /f

  Why WOW6432Node:
    On 64-bit Windows, 32-bit apps write registry to WOW6432Node
    Adobe x86 version used both hives
    Missing this = incomplete cleanup = potential re-registration on boot

STEP 8 — FIX HOSTS FILE
  Reason  : Hosts file was locked by DNS Client service (svchost)
            Direct write fails — must use temp file swap technique
  Commands:
    # Write clean version to temp
    (Get-Content 'C:\Windows\System32\drivers\etc\hosts') |
        Where-Object { $_ -notmatch 'adobe' } |
        Set-Content 'C:\Windows\Temp\hosts_clean.txt'

    # Grant permission on hosts file
    icacls "C:\Windows\System32\drivers\etc\hosts" /grant Everyone:F /c

    # Swap in clean version
    Copy-Item 'C:\Windows\Temp\hosts_clean.txt' `
              'C:\Windows\System32\drivers\etc\hosts' -Force

  Why temp swap:
    DNS Client service (dnscache) keeps hosts file open with a read handle
    Set-Content fails with "file in use" error on locked files
    Writing to a separate file then copying bypasses the lock

STEP 9 — RESTART EXPLORER
  Reason  : Restart Explorer so desktop/taskbar returns
  Command :
    Start-Process explorer.exe

STEP 10 — REBOOT SYSTEM
  Reason  :
    a) Flush all in-memory miner threads/DLLs
    b) Reload registry without Adobe keys
    c) Re-enumerate services (Adobe service binary missing = not started)
    d) Re-enumerate shell extensions (CoreSync DLL missing = not loaded)
    e) Flush DNS cache so hosts changes take full effect
  Command :
    Restart-Computer

STEP 11 — POST-REBOOT VERIFICATION
  Commands:
    # Verify folders gone
    @("C:\Program Files\Adobe","C:\Program Files (x86)\Adobe",
      "C:\Program Files (x86)\Common Files\Adobe",
      "C:\Program Files\Common Files\Adobe") |
        ForEach-Object { Test-Path $_ }

    # Verify hosts clean
    Get-Content 'C:\Windows\System32\drivers\etc\hosts' |
        Where-Object { $_ -notmatch '^#' -and $_.Trim() -ne '' }

    # Verify no miner processes
    $miners = @('xmrig','xmr','miner','nicehash','nanominer','cpuminer')
    Get-Process | Where-Object { $name=$_.Name; $miners|Where-Object{$name -like "*$_*"} }

    # Check GPU usage
    Get-Counter '\GPU Engine(*)\Utilization Percentage' |
        Select-Object -ExpandProperty CounterSamples |
        Where-Object {$_.CookedValue -gt 5}


================================================================================
[5] WHY PROCESS HOLLOWING IS HARD TO DETECT
================================================================================

Normal miner detection looks for process NAMES like:
  xmrig.exe, cpuminer.exe, nicehash.exe

Process hollowing bypasses this by:
  1. Starting a legitimate process (e.g., AdobeUpdateService.exe) suspended
  2. Unmapping the legitimate code from memory
  3. Writing miner code into the same memory space
  4. Resuming the process

Result: Task Manager shows "AdobeUpdateService.exe" but it's actually xmrig.

Tools that CAN detect this:
  > PE-sieve    : pe-sieve.exe --pid <PID>
                  Compares in-memory image vs on-disk binary — mismatch = hollowed
  > Hollows Hunter : hollows_hunter.exe
                  Automated scan of all processes for hollowing
  > Process Hacker : Look for modified PE headers in process memory
  > Volatility  : For memory forensics on a memory dump

[6] QUICK REFERENCE — THREAT HUNTING COMMANDS
================================================================================

--- CHECK HOSTS FILE ---
Get-Content C:\Windows\System32\drivers\etc\hosts | Where-Object { $_ -notmatch '^#' }

--- CHECK RUNNING SERVICES (non-Microsoft) ---
Get-Service | Where-Object {$_.Status -eq 'Running'} |
    Where-Object {$_.DisplayName -notmatch 'Windows|Microsoft'}

--- CHECK NETWORK CONNECTIONS WITH PROCESS ---
Get-NetTCPConnection -State Established | ForEach-Object {
    $p = Get-Process -Id $_.OwningProcess -ErrorAction SilentlyContinue
    [PSCustomObject]@{
        Process       = $p.Name
        RemoteAddress = $_.RemoteAddress
        RemotePort    = $_.RemotePort
        Path          = $p.Path
    }
}

--- CHECK HIDDEN PROCESSES (no file path) ---
Get-Process | Where-Object {
    $_.Path -eq $null -and
    $_.Name -notmatch 'Idle|System|Memory Compression|Registry|Secure System|csrss|smss|wininit'
}

--- CHECK ALL AUTORUN LOCATIONS ---
$keys = @(
    'HKCU:\Software\Microsoft\Windows\CurrentVersion\Run',
    'HKLM:\Software\Microsoft\Windows\CurrentVersion\Run',
    'HKLM:\Software\Microsoft\Windows\CurrentVersion\RunOnce',
    'HKCU:\Software\Microsoft\Windows\CurrentVersion\RunOnce'
)
foreach ($key in $keys) {
    Write-Host "=== $key ===" -ForegroundColor Cyan
    Get-ItemProperty -Path $key -ErrorAction SilentlyContinue |
        Select-Object * -ExcludeProperty PS*
}

--- CHECK SCHEDULED TASKS (non-Microsoft) ---
Get-ScheduledTask |
    Where-Object { $_.TaskPath -notlike '\Microsoft\*' } |
    Select-Object TaskName, TaskPath, State

--- CHECK DEFENDER STATUS ---
Get-MpComputerStatus | Select-Object AMServiceEnabled, AntivirusEnabled,
    RealTimeProtectionEnabled, AMEngineVersion

--- CHECK TOP CPU/RAM PROCESSES ---
Get-Process | Sort-Object CPU -Descending | Select-Object -First 15 Name, Id,
    @{N='CPU_sec';E={[math]::Round($_.CPU,1)}},
    @{N='RAM_MB';E={[math]::Round($_.WorkingSet/1MB,1)}}, Path

--- GPU UTILIZATION (miners hammer GPU) ---
Get-Counter '\GPU Engine(*)\Utilization Percentage' |
    Select-Object -ExpandProperty CounterSamples |
    Where-Object {$_.CookedValue -gt 1} |
    Sort-Object CookedValue -Descending

[8] KEY LESSONS LEARNED
================================================================================

1. ALWAYS CHECK THE HOSTS FILE FIRST
   Modified hosts = immediate red flag. Takes 5 seconds to check.
   Any non-localhost redirects to 127.0.0.1 = investigate immediately.

2. TRUSTEDINSTALLER IS NOT IMPENETRABLE
   takeown + icacls breaks TrustedInstaller protection.
   Malware uses this to hide files — so must you to remove them.

3. KILLING EXPLORER RELEASES SHELL EXTENSION LOCKS
   Shell extension DLLs are loaded into Explorer's address space.
   Explorer kill = DLL unload = file handle released = deletable.

4. ALWAYS CLEAN BOTH REGISTRY HIVES
   HKLM\SOFTWARE\Adobe (64-bit)
   HKLM\SOFTWARE\WOW6432Node\Adobe (32-bit on 64-bit Windows)
   Missing the WOW6432Node hive = incomplete cleanup.

5. LOCKED FILE = USE TEMP SWAP TECHNIQUE
   Write clean content to temp file → copy-replace the locked original.
   Works for hosts file, log files, config files held by services.

6. PROCESS HOLLOWING BYPASSES NAME-BASED DETECTION
   Task Manager process names mean nothing if hollowing is used.
   Must use PE-sieve or Hollows Hunter for memory integrity checking.

7. REBOOT IS MANDATORY — NOT OPTIONAL
   In-memory miner code survives disk deletion.
   Only a full reboot flushes RAM, reloads services, and re-enumerates
   shell extensions cleanly.

8. VERIFY AFTER REBOOT — NOT BEFORE
   Pre-reboot verification can give false positives (in-memory still running).
   Always do final verification AFTER a clean reboot.

9. CRACKED SOFTWARE = HIGHEST RISK INFECTION VECTOR
   "Free" cracked software is the #1 delivery mechanism for:
   - Crypto miners
   - RATs (Remote Access Trojans)
   - Keyloggers
   - Ransomware droppers
   The crack IS the malware. There is no such thing as a "safe crack."

10. BEHAVIORAL INDICATORS ARE OFTEN MORE RELIABLE THAN SIGNATURES
    AV missed this miner. The user noticed it from behavior:
    - RAM always full
    - System hangs under load
    - Laptop always hot
    Behavioral analysis + manual hunting found what AV could not.
